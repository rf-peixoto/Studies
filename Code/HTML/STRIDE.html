Assumptions (predefined)

These assumptions affect threat enumeration. Select only what you can justify.
If an assumption is uncertain, document it explicitly; do not silently “assume secure”.

Security Objectives (predefined)

This does not replace STRIDE; it helps you focus on what “must not fail”.
You can later filter threats by objective in the Review step.
Critical guidance: STRIDE is classification, not risk scoring. If you do not model trust boundaries and privileges (even in simplified form), your STRIDE output will be incomplete and misleading.

Threat Entry Builder
